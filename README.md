# Telegram chatbot
Этот телеграм бот предназначен для общения через API OpenRouter с большими языковыми моделями. На данный момент реализовано общение с текстовыми моделями, которые на вход принимают файлы/фотографии, а на выходе генерируют текст. Далее в планах есть добавить поддержку общения с моделями, способными генерировать изображения. Бот предназначен для личного пользования или совместного использования с друзьями.
## Конфигурация бота.
Для начала работы с ботом необходимо создать `.env` файл со следующей структурой:
```bash
TELE_API_TOKEN="your-telegram-token"
AI_API_TOKEN="your-openrouter-api-token"
BASE_ADMIN="your-telegram-username"
```
где, `TELE_API_TOKEN` - токен вашего телеграм бота, `AI_API_TOKEN` - ваш токен API от OpenRouter, `BASE_ADMIN` - ваш никнейм в телеграм (без @).</br>
Далее вам необходимо изменить `config.py`, имеющий следующую структуру:
```python
from os import getenv
from typing import Any
from dotenv import load_dotenv

load_dotenv()

TELE_API_TOKEN: str = getenv("TELE_API_TOKEN")
AI_API_TOKEN: str = getenv("AI_API_TOKEN")
BASE_ADMIN: str = getenv("BASE_ADMIN")
MAX_MESSAGE_HISTORY = 20
MAX_IMAGE_HISTORY = 4
MAX_PDF_HISTORY = 4
MODELS_IN_USE: list[dict[str, Any]] = [
    {
        "model_name": "openai/gpt-5-mini", "model_type": "text", "input_price": 0.25, "input_cached_price": 0.03, "output_price": 2, "description": "..."
    },
    {
        "model_name": "openai/gpt-5.2", "model_type": "text", "input_price": 1.75, "input_cached_price": 0.175, "output_price": 14, "description": "..."
    },
    {
        "model_name": "anthropic/claude-haiku-4.5", "model_type": "text", "input_price": 5.00, "input_cached_price": 1.00, "output_price": 0.10, "description": "..."
    },
]
ALLOWED_USERS: set[str] = set()
ADMIN_USERS: set[str] = set()
```
где, `MAX_MESSAGE_HISTORY` - максимальное число сообщений, которое сохраняется в диалоге с моделью, `MAX_IMAGE_HISTORY` - максимальное число сообщений, в течение которых сохраняются изображения в диалоге с моделью, `MAX_PDF_HISTORY` - максимальное число сообщений, в течение которых сохраняются отправленные PDF файлы в диалоге с моделью, `MODELS_IN_USE` - это список словарей, содержащих информацию о моделях, `ALLOWED_USERS` - глобальная переменная, хранящая данные о пользователях, которым вы дали доступ к вашему боту, `ADMIN_USERS` - глобальная переменнаяя, содержащая информацию об админах бота (даёт право запускать дополнительные команды). В словарях списка `MODELS_IN_USE` содержится следующая информация: `model_name` - ссылка на модель с OpenRouter, `model_type` - тип модели (`text, image, video, audio`), `input_price` - цена за миллион входных токенов, `input_cached_price` - цена за миллион входных кэшированных токенов, `output_price` - цена за миллион выходных токенов, `description` - описание модели. По умолчанию добавлены модели gpt-5-mini, gpt-5.2, claude-haiku-4.5. Все добавленные модели в `config.py` будут доступны в телеграм боте.</br>

## Начало работы.
Для старта бота запустите файл `main.py`. При первом старте будет создана база данных с помощью SQLite с именем `telebot.db` в папке `db`. В базе данных будут следующие таблицы:
1) users: содержит информацию о пользователях, которым дали доступ к боту. Имеет поля: `id`, `tg_username` - никнейм в телеграм, `has_admin_priviliges` - булева переменная, показывающая права пользователя.
2) user_requests: содержит информацию о запросах отправленных пользователями. Имеет поля: `id`, `user_id` - ключ из таблицы users, `ai_model` - название использованной модели, `money_spent` - стоимость запроса пользователя, `date` - дата запроса.
3) ai_models: информация о добавленных моделях ИИ. Имеет поля: `id`, `model_name` - ссылка на модель с OpenRouter, `model_type` - тип модели (`text, image, video, audio`), `input_price` - цена за миллион входных токенов, `input_cached_price` - цена за миллион входных кэшированных токенов, `output_price` - цена за миллион выходных токенов, `description` - описание модели. По факту в эту таблицу при инициализации БД добавляются данные из переменной `MODELS_IN_USE` в `config.py`.

После запуска бота и нажатии на кнопку `/start` в боте в телеграм откроется основное меню с 4 кнопками:
1) Как пользоваться? - выводит информацию о том, как пользоватья ботом.
2) Узнать траты - выводит информацию о том, сколько вы потратили денег за запросы к ИИ (в долларах).
3) Выбрать модель - позволяет выбрать тип модели (текст, изображение, видео, аудио). При нажатии на тип модели открывается меню с выбором конкретной модели.
4) На главную - позволяет вернуться на главную страницу.

После выбора конкретной модели откроется новый диалог. Отправка простых текстовых запросов происходит посредством отправки сообщений боту. Для добавления файлов или изображений к своему запросу необходимо нажать кнопку **Хочу прикрепить файл(ы)**, отправить в виде сообщений файлы/изображения, дождаться загрузки файлов, затем нажать кнопку **Я закончил**, и наконец написать свой запрос текстом. Такой процесс добавления файлов обусловлен тем, что телеграм считает каждый файл отдельным сообщением. Если вы хотите начать новый диалог (то есть очистить предыдущую историю) с этой же моделью, то нажмите **Новый диалог**. Если вы хотите закончить диалог с моделью, то нажмите **Выйти**.</br>
В диалоге с моделью сохраняются последние 20 сообщений, отправленные PDF файлы и изображения сохраняются в последних 4 сообщениях (для экономии токенов при отправке запросов). Эти параметры можно изменить в `config.py`.</br>
На данный момент есть возможность прикреплять файлы следующего типа: Word (doc, docx), изображения (png, jpg), PDF, текстовые файлы (например, txt, csv, tsv). Поддержка других типов файлов планируется в будущем (Excel).</br>

При ответе большие языковые модели генерируют текст в формате Markdown. В боте реализована функция для конвертации ответа нейросети в Markdown V2, совместимый с Telegram. Конвертация позволяет корректно отображать различные стилистические блоки, в том числе блоки кода. Есть несколько проблем на данный момент:
1. Если блок кода больше 4096 символов, то форматирование блоков кода ломается.
2. Нейросети любят создавать в своём ответе таблицы, но в телеграм нет способа отображать таблицы в сообщениях. Поэтому таблицы в ответах отображаются некорректно.

В будущем, возможно, эти проблемы будут устранены.</br>

Если бот был выключен, когда вы были в диалоге с ботом, то достаточно нажать кнопку **Выйти**, чтобы вернуться на главное меню.
## Проблемы при отправке запросов к модели
Если запрос к модели завершился неудачно, то бот выведет сообщение **Возникла непредвиденная ошибка :(**. Ошибка может появиться из-за слишком большой длиной запроса, перегруженности модели и других проблем. После получения такого сообщения от бота лучше начать новый диалог или выбрать другую модель.

## Команды админа.
Админу доступны команды, которые не могут использовать обычные пользователи. Эти команды необходимо запускать в диалоге с телеграм ботом. Вот перечень команд:
- /help - выводит информацию о всех доступных командах для админа.
- /edit_users - позволяет добавлять или удалять пользователей из БД по никнейму в телеграмме. Полная команда выглядит так: /edit_users add/remove user/admin some_tg_username
- /edit_spendings - позволяет посмотреть или удалить траты пользователей. Для просмотра аргументов команды введите команду в диалоге с ботом.

## Более подробное объяснение работы бота
История диалога пользователя передаётся через FSMContext. В состояние пользователя записываются история сообщений, ссылка на модель и название модели. По факту все эти данные сохраняются в оперативной памяти, а поэтому при отключении бота вся эта информация потеряется.
